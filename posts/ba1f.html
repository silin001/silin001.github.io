  • 文件上传 404 问题
    我（之前）做的（智慧监管中心项目），这个产品的目的是（让客户更直观、更清晰有效的对指标数据结果信息进行解读）。其中有一个（在线会商模块），为了实现（在线视频会议中有个文件上传功能），（在实现文件上传的功能点时遇到了文件上传接口 404 问题）
    首先排查问题：
    1、排查接口路径是否正确；
    2、上传不同类型文件测试是否有同样问题；
    测试发现上传较大一点的文件就会失败、上传小于 10kb 的文件就没问题
    定位到问题后，紧接着我在（知乎、csdn 博客）上找到了参考，联系后端查看 nginx 配置，问题发现出现的原因是（nginx 默认的传输字节数大小默认是 1M，当用户上传超过 10kb 内容后接口就报错）最后修改了 nginx 配置传输字节字段：client_max_body_size 500M成功解决了问题

遇到的代码层面问题及解决方案

  • 计时器问题，a 页面使用后 b 页面还在运行，长时间导致栈溢出？
    在 beforeDestroy 生命周期去清除定时器；
    推荐($once)方案：
const timer = setInterval(() => &#123;
  // 某些定时器操作
&#125;, 500);
// 通过$once来监听定时器，在beforeDestroy钩子可以被清除。
this.$once("hook:beforeDestroy", () => &#123;
  clearInterval(timer);
&#125;);

如果页面使用 keep-aklive 组件缓存后需要在 activated、deactivated生命周期中销毁

  • 请求头 header 自定义参数获取不到？
    排查原因，是因为 nginx 默认将带_下划线的 header 自定义参数中的下划线忽略
    解决办法：将 nginx 的配置 underscores_in_headers 参数设置成on
    该参数默认为 off 会替换下划线，导致获取带有下下划线的参数获取不到

  • echarts 实现中国地图时使用 geoJson数据渲染；地图区域点击事件后返回的区域数据中没有地图区域等级的区分字段？
    在 github 上找一份整理好的各地区 code、level 的集合数据；当地图点击后拿到地区 code 从集合中筛选拿到对应 level

  • includes 替代 if
    多个 if 判断

    return "100";
  &#125; else &#123;
    return "500";
  &#125;
&#125;

做过哪些性能优化

  • vue 项目代码层面优化
    第三方 ui 组件库按需引入
    对于图片过多的页面图片懒加载（如：vue-lazyload 插件）
    路由懒加载（懒加载把页面进行划分，需要时才加载，减少首页加载速度，）
    合理使用 v-if v-show 减少 dom 消耗
    当需要进行数值计算依赖其他数据的时候使用 computed 计算属性因为他有缓存的特性避免每次获取值重新计算
    长列表性能优化（单纯的数据展示列表 使用 object.freeze(obj)来冻结一个对象禁止 vue 劫持监听数据，这能很明显的减少组件初始化时间 ）
    事件及时的销毁（addEventListener()、removeEventListener() 组件销毁时手动进行销毁事件避免内存泄漏）
    vue 使用 keep-akive 组件缓存列表页面
  • 防止重复提交
    会议列表进入会议按钮连续点击后，多次触发该事件？
    使用第三方库 lodash 中的 debounce（防抖函数）解决按钮连续点击问题
  • 引用第三方类库 api，其中初始化方法多处需要使用？
    其中 类似 onReady 方法封装到全局函数中，使用到的地方直接使用全局函数使用

配置层面优化（webpack）

include、exclude 配置项来缩⼩ loader 的处理范围
vue 打包开启 gzip： productionGzip: true, // false 不开启 gizp，true 开启
nginx 开启 gzip 压缩
