开发中问题记录

npm项目依赖问题

  • yarn 安装依赖后启动报错如下
    Could not resolve "pdfast" (mark it as external to exclude it from the bundle)

@antv/g2plot 使用此依赖源码中引用了 pdfast, 重新单独 yarn add ‘依赖库’
一般出现Could not resolve "xxx" 大部分原因是依赖下载失败，就重新下载对应的依赖就行。

vue相关

  • 会议列表进入会议按钮连续点击多次触发事件？
    使用 lodash中的debounce（防抖函数）解决

  • 引用第三方类库api时初始化方法多处需要使用？
    onReady方法封装到全局函数中，使用到的地方直接使用全局函数

  • 文件上传时接口报错404，排查原因后发现nginx默认的传输字节数大小默认是1M，当用户上传超过10kb内容后接口就报错
    解决：nginx配置传输字节字段：client_max_body_size 500M

  • echarts实现中国地图，使用geoJson数据渲染；地图点击事件后返回地图信息中没有地图区域等级区分字段
    手写整理各地区code、level集合；地图点击后拿到地区code去筛选拿到对应level

  • vue 使用 keep-akive 进行缓存组件时用到了计时器，但是被缓存的组件不会销毁，又想清除定时器 这时就用activated、deactivated两个生命周期去清除

  • vite打包部署服务器接口404 需要nginx配置代理！

 # 配置接口 解决404
    location /api {
                proxy_set_header   Host             $host;
                proxy_set_header   x-forwarded-for  $remote_addr;
                proxy_set_header   X-Real-IP        $remote_addr;
                proxy_pass http://123.60.111.103:5000/api;
                # 接口地址，如果在同一个服务器上，可以直接这样写
    }

高德地图使用遇到的问题

  • 高德1.x版本和2.x版本踩坑

      // index.html 引入高德注意事项
  <script type="text/javascript" src="https://webapi.amap.com/maps?v=1.4.15&key=xxx&plugin=AMap.MarkerClusterer"></script>
  // 2.0
  <script type="text/javascript" src="https://webapi.amap.com/maps?v=2.0&key=xxx&plugin=AMap.MarkerCluster"></script>

    高德地图版本为 1.x 和 2.x。 两者部分api语法不同， 且插件名称 部分不同
    （2.0地图渲染比1.x快一些）
    如 点聚合插件：
    2.0: AMap.MarkerCluster
    1.x: AMap.MarkerClusterer

  • 高德地图点聚合2.0写法

      // 初始化聚合点
  function initPointPolymerization (){
      for (let i = 0; i < pointsList.length; i++) {
          if (pointsList[i][LNT] && pointsList[i][LAT]) {
          state.pointPolymerizationMarkers.push({
              data: pointsList[i]['data'], // 自定义的属性数据用于 非聚合点点击时获取
              lnglat: [pointsList[i][LNT].toString(), pointsList[i][LAT].toString()] // 点聚合数据必须为此格式
          })
          }
      }
  }
  // 聚合点自定义
  state.cluster = new AMap.MarkerCluster(state.map, state.pointPolymerizationMarkers, {
      gridSize: 60,
      renderClusterMarker: _renderClusterMarker, // 点聚合样式完全自定义
      renderMarker: _renderMarker //  非聚合点样式设置， 点聚合缩放到最小的marker需要单独设置
  })
      // 设置聚合样式
  function _renderClusterMarker (context) {
      const countLength = state.pointPolymerizationMarkers.length
      const { marker, count } = context
      let div = document.createElement('div')
      div.innerHTML = count
      div.className = 'markersClass1' //添加class
      let size = Math.round(30 + Math.pow(count / countLength, 1 / 5) * 70)
      div.style.width = div.style.height = size + 'px';
      div.style.lineHeight = size + 'px'
      marker.setOffset(new AMap.Pixel(-size / 2, -size / 2))
      marker.on('click', function(e){// 聚合点添加事件})
      marker.setContent(div)
  }
  // 非聚合点样式 + click事件 必须在此函数添加

      const dataArr = context.data[0]
