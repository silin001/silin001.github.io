开发工具使用相关

  • nvm 工具

    nvm list available 显示所有可安装的node版本
nvm install 14.17.6 下载指定版本
nvm uninstall <version> ：卸载指定版本node
nvm use xxx 使用当前下载的某个版本
nvm install stable 安装最新稳定版本node
nvm ls 查看已安装的node版本
nvm current 查看当前版本
nvm arch 显示node是运行在32位还是64位
nvm on：开启node.js版本管理。
nvm off：关闭node.js版本管理。

nvm默认node镜像源是 `https://nodejs.org/dist` 从默认的镜像缘源下载会很慢，所以可以通过更换镜像源加快下载:
`where nvm` 找到nvm安装路径,找到settings.txt文件
将下面这两句话复制到settings.txt 并保存。
`node_mirror: https://npm.taobao.org/mirrors/node/`
`npm_mirror: https://npm.taobao.org/mirrors/npm/`

  • nrm 源管理器
    nrm 是 npm 源管理器，可快速切换 npm 源
    全局安装：
    npm install -g nrm
    nrm ls 查看所有源
    nrm use xxx 使用
    nrm test xxx 速度查看

  • yarn 相关

    // 查看安装的全局包
yarn global list

yarn global bin // 全局安装目录
//这样才是安装到全局环境
yarn global add xxx

// yarn查看所有版本
npm view yarn versions
// yarn更新到最新版
npm install yarn@latest -g

  • 设置淘宝镜像

    
// 查看全局包安装目录
npm config get prefix

npm config set prefix "目录路径" 来设置

npm config set registry http://registry.npm.taobao.org/

// 设置yarn淘宝镜像
yarn config set registry http://registry.npm.taobao.org/

// 查询当前配置的镜像
npm get registry
yarn config get registry

vscdoe 自带同步配置需要登陆微软账号、或者 github 进行同步
使用 Settings Sync 插件

  • github 创建 gists
    我的访问令牌：b5c8881a25f062d29b48932752c9502add834104 （这个令牌只用于访问 gists 文件）
    我的 gistsid：
    vscode 稳定版本专用： 4152e093f7873b65fb574423c7d1cb1c
    vscode 测试版本专用： 8abeb9a8844d2ef7dc283ea82a4a73c3
    两个版本不通用！因为有的插件稳定版有，测试版没有，下载时没有对应插件就会包错还得手动修改远程文件比较麻烦；且测试版本主要是 vue3+TypeScript 的配置主要用于跑 vue3 的个人项目
  • sync 插件使用：
    新设备下载编辑器，安装 Settings Sync 登陆 github
    gist id：xxx
    global Setting: xxx
    command + shift + P（mac）: sync xxx
    下载时可能会报错，编辑器打开 开发人员工具控制台查看具体报错，一般都时远程配置文件的插件找不到，或者出错导致下载中断；
    在 github gists 中找到对应的 entries.json 编辑删除包错插件 重新下载就 ok
  • 查看备份在 github 的配置文件
    登陆 github 点头像选择：your gists， 这里会列出所有不同 gists 的配置文件
    command + shift + P：sync 高级设置里 点击 view

  • Vue Language Features (Volar)插件不生效？
    我在 vue3+ts 项目中安装 Volar 插件后 ts 代码鼠标悬浮变量类型无提示;而关闭 Volar 打开 Vetur 插件是好的但是 vue3 我们还是要使用 Volar，经过自己尝试摸索发现 vscode 版本升级到 1.68.1 版本，Volar 也升级到最新版本一切正常。
    需要注意的是有时 Volar 插件部分功能不生效可能是和 vscode 版本不匹配建议升级到最新稳定版本。
